# Iris Engine 项目功能描述

> 下一代 Rust + WebGPU 无构建前端运行时

---

## 一、项目概览

Iris Engine 是一个革命性的前端运行时，采用 **Rust + WebGPU** 技术栈，**完全消除构建步骤**，直接运行 Vue 3 组件。项目由 17 个核心 crate、1 个 WASM 编译目标和一个 npm 包组成，覆盖了从底层渲染到上层 IDE 集成的完整链路。

### 核心定位

| 维度 | 说明 |
|------|---------|
| **语言** | Rust 1.78+ |
| **渲染** | WebGPU (wgpu 24) 硬件加速 |
| **前端框架** | Vue 3 原生支持 |
| **JS 引擎** | Boa Engine + JetCrab (双运行时架构) |
| **目标** | 零构建 · 高性能 · 桌面/浏览器双端运行 |

## 三、各 crate 详细功能

### 3.1 基础层

| IO 与网络 | 底层文件 I/O 和网络通信 |

#### iris-cssom — CSS 对象模型

|------|---------|

| 哈希标识 | 使用 XXH3 哈希加速选择器匹配 |

### 3.2 渲染层

#### iris-gpu — WebGPU 渲染管线

| 功能 | 说明 |
|------|---------|
| 批渲染系统 (BatchRenderer) | 将所有绘制元素合并为单次 GPU Draw Call，大幅减少渲染开销 |
| 字体纹理图集 (FontAtlas) | GPU 字形缓存，避免重复光栅化，文本渲染性能提升 10-50x |
| 纹理缓存 (TextureCache) | 图片和纹理的 GPU 缓存管理 |
| 脏矩形管理 (DirtyRect) | 只重绘变化区域，节省 50-90% 渲染面积 |
| Canvas 抽象 | 高级绘制接口 (矩形、文本、渐变) |
| 文件监听 (FileWatcher) | 开发期的文件变更监听 |
| 渐变渲染 | 线性渐变和径向渐变的 GPU 实现 |
| WGSL Shader | 自定义批渲染着色器 |

#### iris-layout — 布局引擎

| 功能 | 说明 |
|------|---------|
| 盒模型 | 完整的 CSS 盒模型实现 (margin/border/padding/content) |
| Flex 布局 | Flexbox 完整实现 (flex-direction/wrap/justify-content/align-items/gap) |
| Grid 布局 | CSS Grid 布局 |
| 浮动布局 | float/clear 浮动元素定位 |
| 表格布局 | table/table-cell 布局支持 |
| 定位系统 | static/relative/absolute/fixed/sticky 定位 |
| 虚拟 DOM 树 | 内部 VDOM 树结构、节点操作 |
| 布局缓存 | 缓存热点布局计算结果避免重复计算 |
| 样式计算引擎 | CSS 样式继承、计算、生成 |
| DOM 树操作 | 节点增删改查、遍历、序列化 |
| 事件系统 | 鼠标/键盘/触摸事件的命中测试和分发 |

## 四、双运行时架构

Iris Engine 采用独特的**双运行时架构**，同时支持两条技术路线：

### 路线 A: Boa 原生路线

```
Vue SFC → iris-sfc → iris-js(Boa) → iris-dom → iris-layout → iris-gpu
```

- 使用 Boa Engine 作为 JavaScript 执行引擎
- 原生桌面渲染 (winit + wgpu)
- 适合讲求极致性能的桌面原生应用

### 路线 B: JetCrab 浏览器路线

```
Vue 项目 → iris-jetcrab-engine → iris-sfc → iris-jetcrab(JetCrab) → iris-dom → iris-layout → iris-gpu
```

- 使用 JetCrab 作为 JavaScript 执行引擎
- 完整的 Vue 项目开发服务器 (HMR + 自动刷新)
- 守护进程提供桌面集成 (悬浮图标 + 管理面板)
- 适合浏览器端 Web 开发和日常 Vue 项目开发

### 共享层

两条路线共享以下已稳定的基础层：

- `iris-core` — 内核底座
- `iris-cssom` — CSS 对象模型
- `iris-layout` — 布局引擎
- `iris-dom` — DOM 抽象
- `iris-sfc` — SFC 编译器

## 五、渲染管线流程

```
Vue SFC 文件
    ↓
iris-sfc 编译
    ├── template → 渲染函数
    ├── script setup → TS→JS 转译 + 响应式绑定
    └── style → SCSS/Less/CSS 编译 + scoped CSS
    ↓
Virtual DOM (VNode)
    ↓
动画系统 (AnimationEngine)
    ├── CSS Transition 插值
    ├── @keyframes 关键帧动画
    └── Transform 动画 (translate/scale/rotate/skew/matrix、transform-origin)
    ↓
布局引擎 (Layout)
    ├── 盒模型计算
    ├── Flex/Grid 布局
    └── 定位计算
    ↓
脏矩形管理器 (DirtyRectManager)
    └── 增量重绘区域计算
    ↓
批渲染器 (BatchRenderer)
    ├── 合并 GPU Draw Call
    ├── 字体纹理图集
    └── 渐变/阴影/圆角
    ↓
WebGPU (wgpu)
    ↓
屏幕显示
```
