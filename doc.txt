项目全称
Iris —— Rust+WebGPU 下一代无构建前端运行时引擎

品牌 Slogan
Pupil of Light · 光之瞳

核心定位：
一套完全脱离 Node / 浏览器 DOM / 编译打包、原生双端运行、零编译直接执行 Vue3 + TypeScript、全兼容 Element Plus/Ant Design Vue 等第三方 UI 库的硬件级渲染应用引擎。
运行形态：浏览器 Wasm 模式 + 独立桌面原生模式（双端同一套内核）
核心使命：消灭前端工程化、突破浏览器沙箱限制、给 Vue 生态提供高性能跨端底座

1. 底层内核底座｜Rust 核心基座
语言：纯 Rust 编写，无 GC、内存安全、高性能
基础能力：跨端窗口管理、异步调度、内存池、文件 IO、原生网络栈、缓存系统
跨端适配：
桌面：winit 原生窗口 + Vulkan/Metal/DX12
浏览器：Wasm 编译 + 浏览器 WebGPU API 绑定
核心依赖：wgpu、winit、tokio、reqwest
2. WebGPU 硬件渲染层｜替代原生 DOM 渲染
完全抛弃浏览器 DOM 渲染流水线，全自研 GPU 渲染
基于标准 WebGPU 规范，统一桌面 / 浏览器渲染接口
能力：批渲染、矢量绘制、圆角 / 阴影 / 渐变、纹理图集、字体渲染、图层合成
优势：60fps 稳定渲染、大列表 / 复杂组件无卡顿、CPU 开销极低
3. 布局 & 样式引擎层｜迷你浏览器内核能力
复刻标准浏览器 CSS 体系，对标 Chromium 基础能力：
HTML 解析：html5ever 工业级解析，生成标准 DOM 节点树
CSS 引擎：cssparser 解析、选择器匹配、样式继承、权重计算
布局系统：自研盒模型、Flex、流式布局，对标 W3C 标准
样式挂载：全局样式、Scoped 样式、第三方 UI 库 CSS 全局注入
4. 跨端统一抽象层｜抹平双端差异
统一事件系统：鼠标、键盘、滚动、点击命中检测
统一 BOM/DOM 模拟 API：轻量实现window/document/Event
作用：无缝兼容 Element Plus 等 UI 库所需的浏览器环境 API
无真实 DOM：仅做逻辑模拟，实际绘制全部走 WebGPU
5. JS 沙箱运行时层｜独立隔离执行环境
JS 引擎：QuickJS（轻量高性能、Wasm 友好、Rust 深度绑定）
沙箱隔离：与宿主环境完全隔离，安全隔离脚本
内置运行时：预加载 Vue3 完整运行时（runtime-core /runtime-dom）
模块系统：自研 ESM 解析器，支持import/export、第三方包引入
6. 即时转译层｜核心：零编译能力
核心三大能力，实现源码直接运行：
TypeScript 即时转译
基于 Rust swc，内存内实时 TS→JS，支持泛型 / 装饰器 / TSX
Vue SFC 即时编译
官方 Rust 库解析.vue，自动拆分 template/script-setup/style
Template 实时编译为 Vue 渲染函数
Script 自动 TS 转译
Style 自动解析并入全局样式系统
无构建打包：无 Vite/Webpack/tsc，无node_modules强依赖
7. 上层应用层｜面向开发者
直接运行：.vue / .ts / .tsx 原始源码
生态兼容：完整支持 Element Plus、Ant Design Vue、Naive UI 等 Vue3 生态库
开发模式：源码直接运行、毫秒级热更新、零配置、零依赖安装

附录一 · Iris 视觉识别与 Logo 概念设计
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【设计哲学】
Iris 是眼睛的虹膜——生物界最精密的光学控制器官。它通过瞳孔的缩放与肌理的折射，
精确控制每一束进入视网膜的光线。Iris 引擎正如人眼的虹膜，在 Rust 的坚实底座之上，
精密调度 GPU 渲染管线，让源码之光以最优路径投射为原生界面的清晰影像。

【核心图形符号】
主图形为一只抽象化的虹膜剖面：同心圆环结构，外层是细腻的放射状肌理
（象征 Rust 底层的精密齿轮与晶格），向内渐变为通透的瞳孔。
瞳孔中心不是黑色空洞，而是一束从深处透出的彩虹光谱
（#FF0055 → #FFD700 → #00D4AA → #0080FF），
象征源码经过 Iris 的折射，分解为跨平台运行的多彩光谱。
整体呈现"以机械之眼观天下之色"的赛博生命感。

【色彩系统】
· 主色 Iris Violet：#6B4EE6（虹膜紫，用于品牌主标识，兼具神秘与科技感）
· 辅色 Prism Gold：#FFD700（光谱金，用于高光与重点）
· 科技青蓝 GPU Cyan：#00D4AA（瞳孔光核，用于技术文档与代码高亮）
· 底色 Retina Dark：#0D0D12（视网膜深空黑，深色模式主背景）

【字体建议】
英文标识使用高对比度无衬线字体（如 Satoshi / General Sans / Geist），
字母 "I" 可设计为瞳孔竖线变体，字母 "i" 的点可设计为微型虹膜圆环，
作为 favicon 与品牌符号。
中文标识建议使用现代黑体（如 MiSans / HarmonyOS Sans），
与英文混排时保持略低字重，体现光学仪器的轻盈与精确。

【应用形态】
· 水平组合：同心圆虹膜图标 + IRIS 字标（适用于 GitHub 仓库头图、官网导航）
· 图标独立：纯虹膜瞳孔图形（适用于 crates.io badge、任务栏图标、Wasm 加载动画）
· 动态版本：虹膜肌理随滚动缓慢旋转，瞳孔光谱随鼠标移动产生色差偏移
  （chromatic aberration），模拟真实光学镜头的焦散效果（适用于官网首屏）

附录二 · Crates.io 命名空间与注册规划
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【注册策略】
crates.io 上 `iris` 单字 crate 极大概率已被占用（常见词 + 图像处理库常用名），
因此采用 "核心品牌名 + 模块后缀" 的 Workspace 前缀命名策略。
若 iris-* 系列部分被图像处理领域抢占，则降级为 iri- 紧凑前缀。

【推荐命名方案】

方案 A：iris- 前缀族（首选）
  iris           ← 尝试抢占核心单字 crate（若已被占，改为 iris-engine）
  iris-core      ← Rust 底层内核底座（跨端窗口、内存池、异步调度）
  iris-gpu       ← WebGPU 硬件渲染管线（批渲染、矢量、字体、图集）
  iris-layout    ← 浏览器级布局&样式引擎（HTML/CSS/Flex 解析计算）
  iris-dom       ← 跨端统一抽象层（BOM/DOM 模拟、事件系统）
  iris-js        ← JS 沙箱运行时层（QuickJS 绑定、ESM 模块、Vue3 runtime）
  iris-sfc       ← SFC/TS 即时转译层（swc TS 转译、Vue SFC 编译）
  iris-app       ← 上层应用层入口（桌面 EXE / Wasm 绑定、CLI 工具）

方案 B：iri- 前缀族（紧凑备用）
  若 iris-* 系列部分被占用，可降级为更短前缀：
  iri-core, iri-gpu, iri-layout, iri-js, iri-sfc, iri-app

【注册优先级】
第一梯队（立即注册）：iris / iris-engine、iris-core、iris-app
第二梯队（随开发推进注册）：iris-gpu、iris-layout、iris-js、iris-sfc

【Cargo Workspace 结构预览】
[workspace]
members = [
  "crates/iris-core",
  "crates/iris-gpu",
  "crates/iris-layout",
  "crates/iris-dom",
  "crates/iris-js",
  "crates/iris-sfc",
  "crates/iris-app",
]
resolver = "2"

【命名注意事项】
· 所有 crate 名称需在全小写、短横线连接符规范内
· 注册前务必通过 `cargo search <name>` 确认未被占用
· 建议先注册占位 crate（version 0.0.1）锁定名称，后续迭代发布
